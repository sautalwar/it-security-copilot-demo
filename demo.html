📖 Setting the Scene

It's 8:00 AM at Contoso Corp. CISO Sarah Chen's phone buzzes — the SIEM has flagged anomalous DNS traffic from the production subnet. What follows is a real-time demonstration of how GitHub Copilot transforms incident response, taking us through alert triage, investigation, remediation, infrastructure hardening, and executive reporting — all in a single working day. Each scenario is a live, runnable demo backed by real code, real GitHub Actions workflows, and real security tooling.

1 📡
DNS Exfiltration Alert
Detect and analyze DNS tunneling attempts
✓
💬 Story: Sarah notices a spike in DNS queries to randomized subdomains. The SIEM correlation engine has flagged these as potential data exfiltration via DNS tunneling. Each subdomain is a base64-encoded chunk of stolen data. Time to investigate.
✨

What Copilot Does: Analyzes DNS log patterns, identifies entropy anomalies in subdomain strings, generates detection rules, and produces remediation code to block tunneling attempts.

🎬 Demo Steps
  1. Open VS Code terminal
  2. Run the simulation script:
    $ python scenarios/01-dns-alert/simulate.py
  3. Show the generated vulnerable DNS monitor file in VS Code
  4. Ask Copilot to review the file and suggest fixes
  5. Run the remediation script:
    $ python scenarios/01-dns-alert/remediate.py
  6. Show the diff between vulnerable and fixed versions
  7. Trigger the GitHub Actions workflow:
    $ gh workflow run dns-threat-detection.yml
  8. Open GitHub Actions to show the pipeline running
🎤 Talk Track

"So what we're seeing here is a classic DNS tunneling pattern. The attacker is encoding data in subdomain queries — each of those random-looking subdomains is actually base64-encoded exfiltrated data. Notice the entropy scores Copilot is calculating — legitimate DNS queries typically have an entropy below 3.5, but these are all above 4.0. Now watch what happens when I ask Copilot to analyze this and generate detection rules. It understands network security patterns at a deep level, and within seconds, we have production-ready monitoring code."

📂 Key Files
  • vulnerable-app/dns_monitor.py
  • vulnerable-app/dns_queries.log
