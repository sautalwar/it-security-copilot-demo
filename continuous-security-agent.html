4c. Auto-Remediation with Pull Requests

Automatically generate fixes and open PRs for the security team to review.

auto_remediate.py
#!/usr/bin/env python3
"""Auto-remediation: run fix, commit, and open a PR."""

import subprocess
import json
from datetime import datetime

def create_remediation_pr(scenario_name: str, scenario_num: int, findings: dict):
    date_str = datetime.now().strftime("%Y%m%d-%H%M")
    branch = f"security/auto-fix-{scenario_name}-{date_str}"
    title = f"🔒 Auto-fix: {scenario_name} ({len(findings.get('issues', []))} findings)"

    # Build PR body from findings
    body_lines = [
        f"## 🛡️ Automated Security Remediation",
        f"**Scenario:** #{scenario_num} — {scenario_name}",
        f"**Detected:** {datetime.now().isoformat()}",
        f"**Findings:** {len(findings.get('issues', []))} issues",
        "",
        "### Findings",
    ]
    for issue in findings.get("issues", []):
        severity = issue.get("severity", "medium").upper()
        body_lines.append(f"- **[{severity}]** {issue.get('description', 'N/A')}")

    body_lines += ["", "### Remediation Applied"]
    for fix in findings.get("fixes", []):
        body_lines.append(f"- ✅ {fix}")

    body_lines += [
        "",
        "---",
        "*This PR was automatically generated by the Continuous Security Agent.*",
    ]
    body = "\n".join(body_lines)

    # Git operations
    subprocess.run(["git", "checkout", "-b", branch], check=True)
    subprocess.run(
        ["python", f"scenarios/{scenario_num:02d}-{scenario_name}/remediate.py"],
        check=True,
    )
    subprocess.run(["git", "add", "-A"], check=True)
    subprocess.run(
        ["git", "commit", "-m", f"fix(security): auto-remediate {scenario_name}"],
        check=True,
    )
    subprocess.run(["git", "push", "origin", branch], check=True)

    # Open PR via gh CLI
    subprocess.run([
        "gh", "pr", "create",
        "--title", title,
        "--body", body,
        "--label", "security,auto-remediation,priority:high",
        "--assignee", "@me",
    ], check=True)

    print(f"✅ PR created on branch {branch}")


if __name__ == "__main__":
    # Example usage — in practice, findings come from the scanner
    sample_findings = {
        "issues": [
            {"severity": "critical", "description": "Hardcoded AWS key in src/config.js"},
            {"severity": "high", "description": "S3 bucket policy allows public access"},
        ],
        "fixes": [
            "Replaced hardcoded key with environment variable reference",
            "Updated S3 bucket policy to deny public access",
        ],
    }
    create_remediation_pr("secret-exposure", 5, sample_findings)

They get a pull request with the exact code change, a detailed explanation, severity labels, and a one-click merge button. A PR is 10x easier to deal with than a ticket. And because it goes through the normal code review process, you get an audit trail for free."

5Building a Custom Copilot Extension (Advanced)

What is a Copilot Extension?

  • A custom agent that lives inside GitHub Copilot Chat
  • Users @mention it: @security-agent analyze the latest scan results
  • Has access to repo context and can read security findings, workflow artifacts, and issue history
  • Can trigger remediation workflows and create PRs via the GitHub API

Extension Architecture

💬 Copilot Chat
→
🔌 Webhook
→
🧠 AI Analysis
→
🔧 GitHub API

Backend: Node.js or Python server hosted on Azure / GitHub Codespaces

Example Interaction

U
@security-agent what's our current security posture?
🛡
Based on the last scan (2 hours ago):

✅ DNS monitoring: Clean — no C2 traffic detected
⚠️ Compliance: 2 controls drifted (CIS 5.2.1, 5.2.3)
❌ Secrets: 1 new hardcoded API key found in src/config.js

I've created PR #47 to fix the hardcoded key.
Compliance drift remediation is in PR #48.

Implementation Steps

  1. Create a GitHub App with Copilot Extension capabilities
  2. Build a webhook endpoint that handles Copilot Chat messages
  3. Parse user queries using intent detection (keywords or LLM-based)
  4. Query the repo for latest security scan results (JSON artifacts, workflow runs)
  5. Generate AI-powered responses with remediation suggestions
  6. Optionally trigger workflows or create PRs via the GitHub API

Webhook Handler Skeleton

server.js — Copilot Extension webhook
const express = require("express");
const { Octokit } = require("@octokit/rest");

const app = express();
app.use(express.json());

const octokit = new Octokit({ auth: process.env.GITHUB_TOKEN });

app.post("/copilot/webhook", async (req, res) => {
  const { messages, copilot_references } = req.body;
  const userMessage = messages[messages.length - 1]?.content || "";
  const repo = copilot_references?.find(r => r.type === "repository");

  // ── Intent: Security posture / status ──
  if (/posture|status|overview|summary/i.test(userMessage)) {
    const findings = await getLatestScanResults(repo);
    const summary = generateSecuritySummary(findings);
    return res.json({
      messages: [{ role: "assistant", content: summary }],
    });
  }

  // ── Intent: Remediate / fix ──
  if (/remediate|fix|resolve|patch/i.test(userMessage)) {
    const pr = await triggerRemediation(repo, userMessage);
    return res.json({
      messages: [{
        role: "assistant",
        content: `✅ Created remediation PR #${pr.number}: ${pr.html_url}`,
      }],
    });
  }

  // ── Intent: Investigate ──
  if (/investigate|analyze|kql|query/i.test(userMessage)) {
    const kqlQueries = generateKQLQueries(userMessage);
    return res.json({
      messages: [{
        role: "assistant",
        content: `Here are KQL queries to investigate:\n\`\`\`kql\n${kqlQueries}\n\`\`\``,
      }],
    });
  }

  // Default response
  return res.json({
    messages: [{
      role: "assistant",
      content: "I can help with: security posture, remediation, investigation, and compliance reports. What would you like to know?",
    }],
  });
});

async function getLatestScanResults(repo) {
  // Fetch the latest workflow run artifacts
  const { data: runs } = await octokit.actions.listWorkflowRuns({
    owner: repo.owner, repo: repo.name,
    workflow_id: "continuous-security-agent.yml",
    per_page: 1,
  });
  // Parse findings from the run...
  return runs.workflow_runs[0];
}

async function triggerRemediation(repo, message) {
  // Create a workflow dispatch to trigger remediation
  await octokit.actions.createWorkflowDispatch({
    owner: repo.owner, repo: repo.name,
    workflow_id: "continuous-security-agent.yml",
    ref: "main",
    inputs: { scope: "critical-only" },
  });
  return { number: 47, html_url: `https://github.com/${repo.owner}/${repo.name}/pull/47` };
}

function generateKQLQueries(message) {
  return `SecurityEvent
| where TimeGenerated > ago(24h)
| where EventID in (4624, 4625, 4648)
| summarize Count=count() by Account, Computer, EventID
| where Count > 100
| order by Count desc`;
}

function generateSecuritySummary(latestRun) {
  return [
    "## 🛡️ Security Posture Summary\n",
    "✅ **DNS Monitoring:** Clean — no C2 traffic",
    "⚠️ **Compliance:** 2 controls drifted",
    "❌ **Secrets:** 1 hardcoded key found\n",
    `Last scan: ${latestRun?.created_at || "N/A"}`,
  ].join("\n");
}

app.listen(3000, () => console.log("Security agent listening on :3000"));
